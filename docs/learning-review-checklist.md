# Learning Review Checklist

IssueをCloseする前、または学習ログを書く前に確認します。

## 目標

- 学習テーマが具体的に書かれているか。
- このIssueで何ができるようになればよいか明確か。
- 完了条件が「理解した気がする」ではなく、確認可能な形になっているか。

## 思考過程

- 最初の理解を書いたか。
- わからない点を書いたか。
- 自分で考えた途中式、仮説、説明を書いたか。
- Codexが答えを急ぎすぎていないか。

## 添削

- どこまでは正しかったかを確認したか。
- どこから誤解していたかを確認したか。
- 間違いの原因を1つ以上特定したか。
- 次に同じ間違いを避ける見分け方を書いたか。

## 理解確認

- 自分の言葉で説明できたか。
- 類題または確認問題を1問以上解いたか。
- その結果を記録したか。
- まだ不安な点を分けて書いたか。

## 復習

- 次回復習する内容を決めたか。
- 可能なら復習予定日を書いたか。
- 次のIssue候補がある場合は書いたか。

## Close判断

IssueをCloseしてよい条件:

- 目標に対する理解確認ができた。
- 間違いの原因が記録された。
- 学習ログまたはIssueコメントで振り返りが残った。
- 次の復習または次の課題が明確になった。

Closeしない方がよい条件:

- 答えだけを見て終わっている。
- 自分の考えが書かれていない。
- なぜ間違えたか説明できない。
- 類題で確認していない。
- 次に何を復習するか決まっていない。
