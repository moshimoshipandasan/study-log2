# Issue Label Guide

Issueラベルは、学習カードにつける目印です。
ラベル名は日本語で使います。
小学生にも意味がわかるように、`教科:算数`、`種類:練習する` のような形にしています。

ラベルをGitHubに作成・更新する場合は、次を実行します。

```powershell
npm run sync-labels
```

## ハーネス開始時の必須ラベル

ハーネス開始時は、CodexがまずIssueラベルを整理します。
Issueがどんな学習かをあとから見てもわかるように、最低でも次の3つを付けます。

- `種類:*`: そのIssueがどんな学習か
- `状態:*`: 今どんな状態か
- `公開:*`: Webレポートに載せるか

教科や進め方が判断できる場合は、`教科:*`、`助け:*`、`むずかしさ:*` も付けます。
通常の学習記録は `公開:のせる`、公開したくない内容は `公開:のせない` にします。
`公開:のせる` と `公開:のせない` は同時に付けません。

## 教科

- `教科:算数`: 算数の学習です
- `教科:英語`: 英語の学習です
- `教科:国語`: 国語の学習です
- `教科:理科`: 理科の学習です
- `教科:社会`: 社会の学習です
- `教科:プログラミング`: プログラミングの学習です

## 学習の種類

- `種類:しくみを知る`: 言葉やしくみを知る学習です
- `種類:練習する`: 問題を解いて練習する学習です
- `種類:まちがい発見`: まちがえた理由を見つける学習です
- `種類:ふりかえり`: 前に学んだことをもう一度たしかめる学習です
- `種類:テスト練習`: テストに向けてたしかめる学習です
- `種類:文章を書く`: 文章、作文、レポートを書く学習です
- `種類:ふしぎメモ`: 疑問や確かめたいことを出した学習です

## 今の状態

- `状態:はじめられる`: すぐに学習を始められる状態です
- `状態:考え中`: 何がわからないかを先に見つける状態です
- `状態:見直し中`: 答えや考え方をもう一度見ている状態です
- `状態:できた`: この学習は終わった状態です

## Webレポートに載せるか

- `公開:のせる`: Webの学習レポートに載せます
- `公開:のせない`: Webの学習レポートには載せません

## 必要な助け

- `助け:ヒント`: まずヒントがほしい状態です
- `助け:説明`: 順番に説明してほしい状態です
- `助け:練習`: 似た問題で練習したい状態です
- `助け:見直し`: 自分の答えを見直してほしい状態です
- `助け:覚える`: 覚えて使えるようにしたい状態です
- `助け:先生に聞く`: 先生や保護者にも聞きたい状態です

## 難しさ

- `むずかしさ:やさしい`: 基礎からたしかめる内容です
- `むずかしさ:ふつう`: ふつうの難しさの内容です
- `むずかしさ:むずかしい`: 少しむずかしい内容です

## 注意が必要なもの

- `注意:すぐ答えを見ない`: すぐ答えを見ると考える練習になりにくい内容です
- `注意:テスト関係`: テストに関係する大事な内容です
- `注意:先生や保護者に聞く`: AIだけで決めず先生や保護者にも聞く内容です

## Issueを閉じる目安

- `閉じる目安:説明できた`: 自分の言葉で説明できたら閉じます
- `閉じる目安:見直しできた`: 見直しとふりかえりが終わったら閉じます
- `閉じる目安:似た問題ができた`: 似た問題でたしかめたら閉じます
